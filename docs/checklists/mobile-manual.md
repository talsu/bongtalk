## 자동화 불가 사유 (항목별)

1. **키보드 dodge** — `useKeyboardDodge` 는 `window.visualViewport` resize 를
   구독해 `--m-kb-inset` 을 갱신하는데, Playwright 헤드리스 Chromium 은
   소프트 키보드가 없어 visualViewport 축소를 발생시킬 수 없다(스크립트로
   viewport 를 줄이는 것은 layout viewport 까지 함께 줄어 실제 키보드
   시나리오와 다름). e2e(`keyboard-dodge.polish.e2e.ts`)는 CSS 변수 배선만
   검증하므로 실기기 확인이 유일한 종단 검증이다.
2. **가로 모드 A안** — `orientation-change.polish.e2e.ts` 는 setViewportSize
   기반이라 실제 `orientationchange` 이벤트·회전 중 키보드/주소창 변동을
   재현하지 못한다. A안은 "landscape 에서 데스크톱 셸 노출 = 정책"이므로,
   감사자가 버그로 오인하지 않도록 실기기에서 의도 동작임을 주기 확인한다.
3. **grab 드래그 닫기** — CDP 터치 이벤트로 닫힘 커밋 여부는 자동화돼 있으나
   (`chat-core-sheet.e2e.ts` 등), 추종 지연·fling 속도 판정(0.5px/ms)·스냅백
   모션의 "체감"은 합성 이벤트 타임스탬프로는 판정 불가 — 실기기 감각 확인.
4. **iOS PWA 푸시** — A2HS 설치 플로우와 iOS Web Push 권한 모델은 시뮬레이터
   ·헤드리스 어느 쪽에서도 재현 불가. PRD 알림 도메인 명세(미설치 iOS 는
   인앱 Inbox/배지 폴백)대로 설치 전/후 분기가 맞는지 실기기로 확인한다.
