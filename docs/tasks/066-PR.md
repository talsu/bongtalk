# PR — FR-MN-10 키워드 알림 스캔 (S93 / Task 066)

브랜치 `feat/frmn10-keyword-scan` → develop → main (`--no-ff`).

## 무엇을

메시지 본문에 사용자가 등록한 알림 키워드가 **공백 어절 정확 일치(whole-word·대소문자 무관)**
하면, 그 수신자에게 키워드 멘션(MentionRecord targetType=`KEYWORD`)을 기록하고 라이브 알림
(mention:new · keyword=true)·배지·push 를 발송한다. 스레드 댓글은 제외. 최대 25개 키워드
저장/26번째 400 은 S46/S48 에 이미 존재 — 이 슬라이스는 **스캔/기록/Inbox 연결**을 채운다.

## 설계 핵심

- **새 BullMQ 큐 `mention-scan`** (mention-broadcast/S88b 패턴 미러): 루트 워크스페이스 메시지
  send tx 커밋 후 enqueue → 워커가 잡 시점에 watcher(키워드 보유자) 매칭. 절차: 생존확인 →
  watcher 조회 → `scanKeywords`(순수 함수·어절 정확 일치) → VIEW_CHANNEL 가시성 →
  MentionGateService 게이트(block/mute/DND/NotifLevel·kindFor='direct') → 기존-record dedup →
  `MentionRecord INSERT … ON CONFLICT DO NOTHING RETURNING` → 실삽입분만 mention.received
  outbox(keyword:true). 이후 WS/배지/push 는 기존 outbox-to-ws subscriber 단일 경로(이중경로 회피).
- **Inbox 연결**: me-mentions/me-activity 가 종전 `Message.mentions` JSON 만 봤으나, `OR
EXISTS(MentionRecord targetId=user)`를 **추가형**으로 더해 키워드 멘션을 historical Inbox 에
  노출(ACL 절은 AND 로 보존 — 비가시 private 매치 누출 없음). @role(USER record) 미노출 latent
  갭도 함께 해소(@here/@everyone 은 MentionRecord 미기록이라 무관).
- **마이그레이션**: `20260630`(enum `MentionTargetType` += KEYWORD·additive) + `20260631`
  (부분 인덱스 `UserSettings_keyword_watchers_idx WHERE array_length(keywords,1)>0` · perf).
